# Planejamento do Projeto E-commerce Backend

Este documento serve como o roadmap de desenvolvimento do seu projeto, focado em demonstrar senioridade técnica para portfólio.

---

## 🚀 **Módulos de Negócio**
1. **Catálogo de produtos:** CRUD completo, upload de imagens, filtros e paginação.
2. **Gerenciamento de usuários:** Autenticação JWT, perfis (ADMIN/USER), recuperação de senha.
3. **Carrinho de compras:** Adição/remoção de itens e cálculo de totais.
4. **Processamento de pedidos:** Fechamento de pedidos, histórico e status.
5. **Sistema de pagamento:** Simulação de checkout com eventos assíncronos.
6. **Avaliações de produtos:** Sistema de reviews e notas (Estrelas).

---

## 🛠️ **Módulos de Infraestrutura & Boas Práticas (Diferencial Sênior)**

1. **Documentação Automática:**
   - [ ] Implementar **Swagger/OpenAPI (Springdoc)** para documentação interativa da API.
2. **Gestão de Banco de Dados:**
   - [ ] Substituir `ddl-auto=update` por **Flyway/Liquibase** para migrações versionadas.
3. **Dockerização & DevOps:**
   - [ ] Criar **Dockerfile** multi-stage para a aplicação.
   - [ ] Configurar **Docker Compose** (Java + MySQL + Redis).
   - [ ] Configurar **GitHub Actions** para CI/CD (Build & Tests).
4. **Resiliência & Performance:**
   - [ ] Implementar **Caching com Redis** para consultas pesadas (ex: categorias).
   - [ ] Adicionar **Soft Delete** em entidades críticas (Produtos/Pedidos).
5. **Observabilidade:**
   - [ ] Ativar **Spring Boot Actuator** para monitoramento de saúde da API.
   - [ ] Configurar logs estruturados com SLF4J/Logback.
6. **Testes de Alta Qualidade:**
   - [ ] Cobertura de testes unitários com JUnit 5 e Mockito.
   - [ ] Testes de integração reais usando **Testcontainers** (MySQL real no Docker).

---

## 🔄 **Processo de Desenvolvimento Sugerido**

1. **Semana 1**: Setup + Entidades básicas (Produto, Categoria) + Flyway inicial.
2. **Semana 2**: CRUD completo com validações + Paginação + Upload de Imagens.
3. **Semana 3**: Autenticação, Autorização e Documentação (Swagger).
4. **Semana 4**: Carrinho, Pedidos e Eventos de Pagamento.
5. **Semana 5**: Testes Unitários e de Integração (Testcontainers).
6. **Semana 6**: Docker, Caching (Redis) e Observabilidade (Actuator).
